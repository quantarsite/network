Methodology
Parameter Detail
Benchmark type Signature verification throughput — parallel transaction processing, simulating consensus-layer load without network overhead.
AND-composition Each transaction requires a valid ML-DSA-87 signature AND a valid SPHINCS+ signature. Both are verified sequentially per transaction; transactions are distributed across threads.
Sample size 10 independent runs per configuration. Results averaged. Outliers (±2σ) excluded.
Parallelism Rayon thread pool (Rust). Each core processes an independent transaction batch. No shared mutable state between threads.
Keys Pre-generated keypairs, excluded from timing. Benchmark measures pure verification cost.
Implementation Rust. pqcrypto-dilithium for ML-DSA-87 · pqcrypto-sphincsplus for SPHINCS+.
