Protocol Overview
A transaction is valid only when both schemes verify independently.

HYDRA-X is the dual post-quantum signature protocol enforced at the consensus layer of Quantar Network. It is not an application-layer wrapper or an optional module. Every block, every transaction, and every validator attestation must carry a valid HYDRA-X composed signature to be accepted by the network.

The AND-composition means an attacker must break two mathematically unrelated paradigms simultaneously. A breakthrough against lattice cryptography does not affect the hash-based component, and vice versa. The composed scheme is at least as strong as the stronger of the two components.

HYDRA-X Verification Rule
Ver_L(pk_L, tx, σ_L) = 1   AND   Ver_H(pk_H, tx, σ_H) = 1
Ver_L = ML-DSA-87 / Dilithium-5  ·  NIST FIPS 204  ·  Lattice / Module-LWE
Ver_H = SLH-DSA-SHAKE-256f  ·  NIST FIPS 205  ·  Hash-based / Stateless

Security Stack
Five cryptographic layers. Zero single point of failure.

HYDRA-X is structured in five independent layers, each addressing a distinct attack surface. All five are enforced at the protocol level — not optional, not application-dependent.

L3⏱ ENFORCED
Temporal Expiration Engine
Mandatory Key TTL
Enforced at consensus level — not optional. The protocol rejects signatures past TTL. No exceptions, no developer discipline required.
TTL Mechanism Each keypair carries a block_expiry on-chain. Protocol rejects signatures past TTL. Migration MigrateKey(old_pk, new_pk, σ) — signed by both old keypairs. Atomic renewal.

Validator nodes verify n dual-signatures in batch, amortizing SPHINCS+ overhead across all block transactions.

Formal Security
EUF-CMA / EUF-qCMA security bound.

The AND-composition proof reduces the security of HYDRA-X to the hardness of the two underlying problems. The composed advantage is bounded by the sum of the component advantages plus a negligible term.

Formal Security Bound
Adv(HYDRA-X)  ≤  Adv(ML-DSA-87)  +  Adv(SPHINCS+)  +  Advcoll(SHA3)  +  negl(λ)
